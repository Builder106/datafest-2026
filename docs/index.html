A single question — "Has lack of transportation kept you from medical appointments?" — identifies a cohort with ~3× odds of acute-care need, independent of age. This site documents the analysis. The reproducible R + DuckDB pipeline lives on GitHub.

What this is

Team 13's submission to ASA DataFest 2026 at Wesleyan University, April 17–19, 2026. The data sponsor was Stormont Vail Health, which released a longitudinal EHR sample joined to a 12-domain social-determinants questionnaire.

I asked: Do patients who report a transportation barrier experience measurably different healthcare journeys than otherwise similar patients? If yes, the transport question itself becomes a screening signal — an upstream variable the health system can act on with rides, tele-visits, or case management.

Key findings

Outcome No barrier Transport barrier Effect
ED visits per person-year 0.48 1.94 4.0× crude
Inpatient admits per person-year 0.22 0.70 3.2× crude
Any ED visit (prevalence) 43% 68% OR 3.17 (95% CI 2.93–3.43)
Any inpatient admit (prevalence) 35% 63% OR 3.49 (3.23–3.77)

The logistic model is outcome ~ transport + age + sex, fit on n = 58,639 screened patients. Barrier patients are younger (median 51 vs 61), ruling out an age artifact. The ED-return gap also persists inside every chronic-disease cohort we examined (HTN 33% vs 15%, T2D 40% vs 17%, CKD 37% vs 21%, AFib 34% vs 24% — 180-day ED return after index encounter).

Reproducing the analysis

The raw EHR data is not in the repository (data-use agreement with Stormont Vail). The pipeline is reproducible against any DataFest 2026 CSV bundle dropped into DataFest 2026 - Data Challenge/Data/2026-ASA-DataFest-Data-Files/.

# 1. R packages (one-time, into ~/R/datafest_libs)
Rscript -e 'install.packages(c("data.table","duckdb","DBI","dplyr","tidyr","stringr","lubridate","ggplot2","scales"), lib="~/R/datafest_libs")'

# 2. Drop the official DataFest 2026 CSV bundle into place,
#    then run the pipeline (builds ~/.datafest_cache/datafest.duckdb on first run)
Rscript analysis/R/run_all.R

# 3. Smoke-test the outputs
Rscript analysis/tests/smoke_test_outputs.R

Full run order and outputs are documented in analysis/README.md.

Caveats

The data is observational and the screening sample is non-random. The major caveats:

  • Only ~6% of patients (61,052 / 947,685) have any SDOH answer; absolute prevalences should not be projected to the full system.
  • 20% of encounter rows carry a PrimaryDiagnosisKey not found in the diagnosis lookup — the chronic-disease cohort is a lower bound.
  • 65% of patients have no parseable FIPS code, so no geographic model was fit.
  • DepartmentType is *Unknown for 71% of rows; setting analyses use boolean encounter flags (IsEdVisit, IsInpatientAdmission) instead.
  • Patient journeys are left- and right-censored at the 2022-01-01 / 2025-12-31 window; rates are annualized by observed follow-up.
